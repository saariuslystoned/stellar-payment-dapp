# Smoky Coins Tokenomics

## Overview
The Smoky Coins ecosystem utilizes a dual-token model to separate governance/asset value from loyalty rewards.

---

## 1. SMOKY Token (Governance / Asset)
**Asset Code**: `SMOKY`
**Type**: Fixed Supply, Asset-Backed Target
**Home Domain**: (Pending)

### Supply Model
- **Total Supply**: 100,000,000 (Fixed)
- **Issuer Account**: Locked permanently after minting.
- **Valuation Goal**: 1 SMOKY = $1.00 USD ($100M Market Cap)

### Distribution
| Allocation | Amount | Percentage | Description |
|------------|--------|------------|-------------|
| **Public Sale** | 49,000,000 | 49% | Available for purchase by users. |
| **Founder Reserve** | 51,000,000 | 51% | Held by founder (Exit strategy). |

### Utility
1.  **Governance**: Voting weight for protocol decisions.
2.  **Tier System**: Holding more SMOKY unlocks higher reward tiers.
3.  **Staking**: Stake SMOKY in Blend pools to earn ZMOKE yield.

---

## 2. ZMOKE Token (Rewards)
**Asset Code**: `ZMOKE`
**Type**: Elastic Supply, Utility Token
**Home Domain**: (Pending)

### Supply Model
- **Minting**: Elastic. Minted programmatically when users earn rewards.
- **Burning**: Burnt when redeemed for discounts/perks.

### Earning Mechanics
1.  **WooCommerce Purchases**: 10 ZMOKE per $1 spent.
2.  **Signup Bonus**: 100 ZMOKE for new account registration.
3.  **Staking Yield**: Earned via Blend pools (rate dynamic).

### Redemption
- Users burn ZMOKE to redeem for:
    - Product discounts
    - Exclusive access
    - Digital downloads

### Onboarding
- **Claimable Balance**: New users receive their first ZMOKE via Claimable Balance, which automatically establishes the ZMOKE trustline upon claiming.
